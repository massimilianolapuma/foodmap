# Session 0003 — 2026-05-31 — p11-replace-recipe

**Branch:** `feat/46-replace-recipe` · **Phase:** `P11` · **Issues:** #46

## Goal
Let the user swap a single recipe in a meal plan, choosing among a few similar
alternatives, while keeping the rest of the plan and the expiring-first focus.

## What changed
- `MealPlannerAIService` gains `alternatives(for:from:profile:count:)` with a
  default `[]` implementation so existing mocks keep compiling.
- `RuleBasedMealPlanner.alternatives` returns expiring-first alternatives that
  keep the meal slot (type/day) and exclude the current primary product.
- `FoundationModelsMealPlanner.alternatives` delegates to its fallback.
- `MealPlannerViewModel` exposes `alternatives` / `isLoadingAlternatives`,
  `loadAlternatives(for:)`, and `replace(_:with:)` (persists the updated plan).
- `MealDetailView` adds a Replace toolbar action + `ReplaceRecipeView` sheet
  (loading / empty / list states); `MealPlannerView` passes the view model.
- Localized strings (en/it) for the replace flow.
- Tests: `RuleBasedMealPlannerTests` (slot/priority/exclusion) and
  `MealPlannerViewModelTests` (loadAlternatives + replace).
