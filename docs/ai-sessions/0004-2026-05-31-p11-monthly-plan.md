# 0004 — P11: Monthly meal plan horizon (#47)

Date: 2026-05-31
Branch: `feat/47-monthly-plan` · PR: #75 · Closes #47

## Goal

Allow generating a **monthly** meal plan in addition to single-day / 3-day / week,
keeping expiring-items prioritization and shopping-list generation, and staying
performance-conscious for the larger horizon.

## Changes

- `Domain/ValueObjects/DietEnums.swift`: `MealPlanType` gains a `.month` case and
  now centralizes horizon logic with `dayCount` (1/3/7/30) and `displayName`,
  removing the duplicated `switch` statements that lived in each planner.
- `Data/DataSources/RuleBasedMealPlanner.swift`: uses `planType.dayCount`; adds a
  `"Monthly plan"` title. Expiring-first sorting and shopping-list generation are
  horizon-agnostic and apply automatically.
- `Data/DataSources/FoundationModelsMealPlanner.swift`: uses `planType.dayCount`
  (removed the redundant private wrapper) and adds a `"Monthly plan"` default title.
- `Features/MealPlanner/MealPlannerView.swift`: Horizon picker offers a **Month**
  option.
- `Resources/{en,it}.lproj/Localizable.strings`: `"Month"` entries.
- `FoodMapTests/RuleBasedMealPlannerTests.swift`: `testDayCountMatchesPlanType`
  now covers `.month` (30 days → 60 meals).

## Performance note

The generation loop is linear in days × 2 slots, so a 30-day horizon is just a
larger constant — no heavy per-day work was added.
